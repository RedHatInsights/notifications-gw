= notifications-gw
:toc: auto
:toclevels: 3

Notifications Gateway is a Red Hat Insights microservice that receives notification payloads via REST, validates them against `notifications-backend`, and forwards them to Kafka (`platform.notifications.ingress`). It acts as the public entry point for the notifications platform -- there is exactly one Kafka producer channel and no consumers. The gateway does not own any database or persistent state.

The service is built with *Java 21* and *Quarkus 3.30.5*, packaged as a JVM container on UBI 9, and deployed via Clowder on OpenShift. It is licensed under the Apache License 2.0.

== Getting Started

=== Prerequisites

* *JDK 21* (e.g., OpenJDK, Adoptium)
* *Docker* or *Podman* (for Kafka in dev mode)

=== Running the application in dev mode

Start Kafka first using the included Docker Compose file, then launch Quarkus in dev mode with live reload:

[source,shell]
----
docker-compose up -d
./mvnw quarkus:dev
----

The dev server starts on port *8086* (see `application.properties`).

=== Building and running the application

Package the application:

[source,shell]
----
./mvnw package
----

This produces a fast-jar in `target/quarkus-app/`. Run it with:

[source,shell]
----
java -jar target/quarkus-app/quarkus-run.jar
----

To build an uber-jar instead:

[source,shell]
----
./mvnw package -Dquarkus.package.type=uber-jar
----

=== Running tests

Tests require *no external services* -- `TestLifecycleManager` starts MockServer and switches Kafka to an in-memory connector automatically. Docker is not needed.

[source,shell]
----
# Full build + all tests (matches CI)
./mvnw clean package --no-transfer-progress

# Tests only (faster iteration)
./mvnw test

# Single test class
./mvnw test -Dtest=GwResourceTest

# Single test method
./mvnw test -Dtest=GwResourceTest#testNotificationsEndpoint

# Enable MockServer logging for debugging
./mvnw test -Dmockserver.logLevel=WARN
----

See link:AGENTS.md[AGENTS.md] for additional test tips including cache invalidation and parameterized test patterns.

== Configuration

=== Key Application Properties

All application-specific properties use the `notifications.*` prefix. Key settings include:

* `notifications.bulk-caches.enabled` -- Toggle between per-request REST validation and locally cached bulk validation
* `notifications.allow-list.enabled` / `notifications.allow-list.org-ids` -- Org ID allow-list for staging environments
* `notifications.kafka-callback-timeout-seconds` -- Kafka ack/nack callback timeout
* `notifications.emails.internal-only.enabled` -- Restrict recipient emails to internal addresses

=== Clowder Config Source

This project uses the link:https://github.com/RedHatInsights/clowder-quarkus-config-source[Clowder Quarkus Config Source]. To point it at a different config file than the default `/cdappconfig/cdappconfig.json`:

[source,properties]
----
clowder.file=/path/to/file.json
----

|===
| Environment | Port

| Dev (`application.properties`)
| 8086

| Production (`clowdapp.yaml` via `QUARKUS_HTTP_PORT`)
| 8000

| 8080

=== Non-application Root Path

Health (`/health/*`), metrics (`/metrics`), and OpenAPI (`/openapi.json`) are served at `/` -- *not* under `/q/`. This is set by `quarkus.http.non-application-root-path=/`.
